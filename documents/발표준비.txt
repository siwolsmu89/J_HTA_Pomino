[1] 초기 화면

1. 홈 화면
- 각각의 페이지와 연결되는 통로
- 첫 노출 페이지 : 가장 자주 사용될 법한 기능들을 전진 배치

- 클릭 되는 부분들 한번 언급하고 넘어가기

2. 메뉴 화면
- 전체 피자 목록을 보여주는 페이지
- 간단한 정보를 직관적으로 제시
- 이벤트 뱃지를 클릭하면 이벤트 상세 화면으로 넘어가고 할인 가격이 적용

- 모달 창을 사용하여 화면 전환 최소화
	- 돋보기 클릭시 상세 정보 제공
	- 영양성분 및 알레르기 유발 성분 표시창

3. 가입 및 로그인, 배송지 등록
- 주문을 위해서는 회원가입 및 로그인이 필요함
- 로그인 후 주문창을 넘어가기 위해서는 배송지를 등록하고 선택해야 함
- 이미 있는 아이디는 회원가입 방지
- 회원 탈퇴한 아이디로 로그인을 시도하면 회원 가입 페이지로 이동

4. 배송지 선택
- 로그인 후 최초의 주문 절차를 진행할 경우 배송지를 선택해야 함
- 배송지 선택 시 적절한 매장 주소가 없으면 주소지 선택 불가
- 배송주소 등록 : 별도의 화면 이동 없이 모달창으로 입력하도록 작성
- 선택된 배송지 번호는 세션에 저장

5. 상세 피자 화면
- 위에서 선택한 주문내역이 하단 고정바에 표시되어 손쉽게 주문내역 확인가능
- 각 항목의 계산은 javascript function을 사용하여 onclick, onchange 이벤트 발생에 연동시킴
- 주문하기 클릭시 db에 해당 주문정보를 장바구니 타입으로 저장하여 
이후 일일이 요청객체에 값을 넣어주지 않아도 편하게 값을 불러올 수 있도록 작성

6. 장바구니 화면
- 간단한 상품 정보 확인 및 최종 수량 변경 가능
- 메뉴 추가를 원하면 추가 메뉴를 등록할 수도 있음

7. 주문하기 화면
- 장바구니에서 최종 수정한 DB의 정보들을 가져와서 최종 금액을 표시
- 회원 등급 또는 매장 할인이 적용되는 지점

- 상세 피자 화면부터 주문하기까지의 과정은 서로 연계성도 높고, 전체 테이블의 절반 가까이 되는 테이블들을 사용하는 부분
- 금액과 관련된 부분이기도 하고, 피자 홈페이지에서 핵심적인 기능이라고 할 수 있으므로 공을 많이 들임

8. 결제 완료 및 주문 정보 조회
- 결제 완료 시 주문 내역 조회 화면으로 이동
- 주문 번호 역순으로 정렬하여 최근 주문부터 확인 가능
- 재주문, 퀵오더 등록 등의 단순한 기능은 목록에서 즉시 사용 가능
- 상세보기 이동 시 자세한 주문 내역을 확인할 수 있음
- 주문 상태는 progress bar를 사용하여 직관적으로 확인하도록 작성

9. 내 등급 보기
- 현재 등급, 남은 주문 횟수, 등급 혜택, 총 주문 금액 등을 확인할 수 있음

10. 1:1문의
- 문의 모달 창에서 지역을 선택하면 해당 지역에 존재하는 매장만 표시하도록 ajax 사용
- 제목 클릭 시 수정, 삭제, 답변 및 상세 내용 조회 가능
- 글 삭제 시 제목 표시만 변경되도록 처리

11. 정보수정
- 사용자 정보 수정 폼
- 이름과 아이디는 변경 불가능
- 탈퇴 절차도 여기서 처리함

12. 회원탈퇴
- 탈퇴 사유 선택, 탈퇴 사유 상세 내용 작성 후 탈퇴 처리되도록
- 탈퇴시 얼럿 창을 하나 띄운 뒤 로그아웃 및 홈 화면으로 이동

13. 퀵오더 및 재주문 (빠르게 확인)

14. 이벤트에서 주문하기 (빠르게 확인)

15. 매장검색 한번 쓱 보고 지나가기
